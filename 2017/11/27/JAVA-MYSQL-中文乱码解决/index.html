JAVA MYSQL 中文乱码解决

发表于 2017-11-27 | 分类于 编程语言学习

令人头疼的问题

今天在写Java项目的时候，再次遇到了向Mysql写如中文出现乱码的问题。纠结了一个多小时，总算解决了。
可能出现乱码的原因有以下几种：

MySQL的默认编码是latin1

针对这个问题需要修改MySQL的默认的编码，在Win下可以通过修改 my.ini 文件来设置。在Mac下没有找到这个文件，我们可以使用 [这篇博客] 的方法来创建一个MySQL的配置文件。或者我们可以通过MySQL命令来设置。

设置之后我们来看

奇怪的是设置了编码之后仍然有问题

那就是可能出在了连接层
我们在连接数据库的时候需要在连接url上加上一句话

useUnicode=true&characterEncoding=utf-8

还是不行。
找了好多办法，最后发现，应该在建立数据表的时候，设置表的默认字符集。

就是这句话：

charset utf8 collate utf8_general_ci
