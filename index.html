递增的三元子序列

发表于 2019-03-23

递增的三元子序列

Given an unsorted array return whether an increasing subsequence of length 3 exists or not in the array.

Formally the function should:

Return true if there exists i, j, k 
such that arr[i] < arr[j] < arr[k] given 0 ≤ i < j < k ≤ n-1 else return false.

Note: Your algorithm should run in O(n) time complexity and O(1) space complexity.

Example 1:

Input: [1,2,3,4,5]
Output: true

Example 2:

Input: [5,4,3,2,1]
Output: false

思路

对于这道题来说，很容易想到的是可以通过一个三重循环遍历来实现。但是这样做太慢。

另一个容易想到的思路是，使用两个指针，分别从前往后找最小和从后往前找最大。这样，只要最大值比最小值靠后则一定是存在递增三元序列的。

这样的方法是不行的

乍一想貌似正确，但是仔细想想，递增三元组并不要求一定是由整个数组的最大值和最小值组成的。例如：

2, 4, -5, -3, -2, 2, 0

整个数组的最大值是4，最小值是-5，4位于-5之前，但是却存在递增三元组 -5, -3, -2。

动态规划

如果从动态规划的角度来考虑这个问题，如果我找到了三元组中的中间的数，那么只要在剩余的数组中存在比中间数更大的数就必然存在三元组了。

那什么叫做找到了中间的数呢，其实就是找到了次小的数。也就是说我们只需要记录最小和次小的两个数就可以了。在实现过的过程中，遍历数组根据大小关系更新最小和次小的数字：

  • 如果当前数字小于最小的数，则把最小的数更新为当前数。
  • 如果大于最小，小于次小，更新次小。
  • 如果大于次小，则返回true。

动态规划方法往往会在所记录的数字中包含更多的信息，在本题中，次小数不仅仅是当前遍历过的这一部分数组中第二小的数，同时还隐含了在这个数之前，存在一个比他小的数。因此只要有比他大的数字，就存在递增三元组。
